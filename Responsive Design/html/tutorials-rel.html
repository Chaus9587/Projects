Relative units

Introduction

You may have learnt some common absolute units in css such as px, cm, etc.


However, those units may become a problem when you want to achieve a highly responsive webpage as you may need to use a lot of @media to change the size of some text or container.
But if you don't to use a lot of @media query, the relative units will come in handy.

Examples of relative units

1: % units

It is relative to the width/ height of the parent element.


HTML:

<div class="parent-container">
    <div class="child-container">
        Lorem30 (Simplification)
    </div>
</div>

CSS:

.parent-container {
    width: auto;
    border: 1px solid black;
}
.child-container {
    width: 50%;
}

Output:


If change the width of the child container to 75%:


CSS:

.child-container {
    width: 75%;
}


You can also use the % unit on the font-size property.


HTML:

<section>
    <div>
        Lorem4
        <div>
            Lorem4
            <div>
                Lorem4
                <div>
                    Lorem4
                </div>
            </div>
        </div>
    </div>
</section>

CSS:

section {
    font-size: 16px;
}
p {
    font-size: 125%;
}

Output:



2: em/rem units.

em is similar to percentage units. (decimal instead of percentage) em is relative to the parent element where rem is relative to the root of the webpage. (usually <html> element)


HTML: em

<div>
    Lorem4
    <div>
        Lorem4
        <div>
            Lorem4
            <div>
                Lorem4
            </div>
        </div>
     </div>
</div>

CSS:

p {
    font-size: 1.25em;
}

Output:


As metioned above, rem is relative to the root element of the webpage.


HTML: rem

<div>
    Lorem4
    <div>
        Lorem4
        <div>
            Lorem4
            <div>
                Lorem4
            </div>
        </div>
    </div>
</div>

CSS:

html {
    font-size: 16px; /* Default */
}
div {
    font-size: 1.25rem;
}

Output:


3: vw/vh units

It is relative to the viewport size.(viewport width/viewport height)


HTML:

<div class="container">
  Lorem300
</div>

CSS:

.container {
  width: 70vw;
  height: 70vh;
  overflow: scroll;
}

Further uses of relative units

Although those relative units are powerful, there are some ways to make it even more powerful.

1: calc()

calc() function allows you to perform calculations for specifying a property value.

See the examples below.


HTML:

<div class="div1">div1</div>
<div class="div2">div2</div>
<div class="div3">div3</div>
<div class="div4">div4</div>

CSS:

div {
    display: flex;
    justify-content: center;
    align-items: center;
}
.div1 {
    background-color: skyblue;
    width: calc(80px + 2vw);
    height: calc(80px + 2vh);
}
.div2 {
    background-color: cornflowerblue;
    width: calc(80px + 4vw);
    height: calc(80px + 4vh);
}
.div3 {
    background-color: lightcoral;
    width: calc(120px + 6vw);
    height: calc(120px + 6vh);
}
.div4 {
    background-color: coral;
    width: calc(120px + 3.5vw);
    height: calc(120px + 3.5vh);
}


The same concept applies to the font-size, margin, padding and similar properties.


HTML:

<p class="p1">Lorem4</p>
<p class="p2">Lorem4</p>

CSS:

.p1 {
    font-size: calc(20px + 4vw);
}
.p2 {
    font-size: calc(4vw - 20px);
}

2: clamp()

clamp() function is similar to calc(), but with a minimum and maximum value.


Syntax:

clamp(|minimum|, |preferred|, |maximum|)

Some examples are shown below.


HTML:

<div class="div1">div1</div>
<div class="div2">div2</div>

CSS:

div {
    display: flex;
    justify-content: center;
    align-items: center;
}
.div1 {
    background-color: cornflowerblue;
    width: calc(6vw - 20px);
    height: calc(6vw - 20px);
}
.div2 {
    background-color: lightcoral;
    width: clamp(80px, 6vw - 20px, 250px);
    height: clamp(80px, 6vw - 20px, 250px);
}


You can see the difference when there is a minimum & maximum value, it prevents a responsive element from getting too small/ big.

This works the same on text elements.
